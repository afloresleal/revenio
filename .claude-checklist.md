### PASO 5: Escribir Tests (MANDATORY)

**REGLA:** Ningún código se mergea sin test.

**Para cambios en DB/tipos, SIEMPRE escribir:**

```typescript
// apps/api/test/[feature-name].test.ts

// Test 1: Verificar propagación de campos
test('resolveGhlCampaign propagates new field from DB', async () => {
  // Mock prisma
  const mockCampaign = { nuevoField: 'test-value' };

  const result = await resolveGhlCampaign('test-id');

  assert.equal(result.nuevoField, 'test-value',
    'must propagate nuevoField from DB to returned object'
  );
});

// Test 2: Verificar lógica de negocio
test('evaluator uses campaign-specific value not global', () => {
  const campaign = { nuevoField: 'campaign-value' };
  process.env.GLOBAL_VALUE = 'global-value';

  const result = evaluate(campaign);

  assert.equal(result.value, 'campaign-value',
    'must use campaign value, not global env'
  );
});

// Test 3: Verificar backward compatibility
test('falls back to global when field is null', () => {
  const campaign = { nuevoField: null };
  process.env.GLOBAL_VALUE = 'global-value';

  const result = evaluate(campaign);

  assert.equal(result.value, 'global-value',
    'must fall back to global when campaign field is null'
  );
});
```

**Correr tests:**
```bash
npm -w apps/api test apps/api/test/[feature-name].test.ts
```

**Decirle al usuario:**
```
✅ Test creado: apps/api/test/call-window-propagation.test.ts
✅ 3 tests escritos:
  - Propagación de campos ✅
  - Lógica usa valor correcto ✅
  - Backward compatibility ✅
✅ Todos los tests pasan
```
